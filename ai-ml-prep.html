01 Core ML concepts
Fundamentals — always asked
Basic What is the difference between supervised, unsupervised, and reinforcement learning? ▶
Answer
  • Supervised learning — you have labelled data (input + correct output). The model learns to map inputs to outputs. Examples: classification (spam/not spam), regression (predict house price). Your intrusion detection thesis is supervised — network traffic labelled as attack or normal.
  • Unsupervised learning — no labels. The model finds patterns on its own. Examples: clustering customers, anomaly detection, dimensionality reduction.
  • Reinforcement learning — an agent learns by trial and error, getting rewards or penalties. Examples: game-playing AI, robotics.

Basic What is overfitting and how do you prevent it? ▶
Answer

Overfitting is when a model learns the training data too well — including its noise — and fails to generalise to new, unseen data. High accuracy on training data, poor accuracy on test data.

How to prevent it:

  • More training data — the most effective fix
  • Regularisation — L1/L2 penalties that discourage overly complex models
  • Cross-validation — validate on multiple data splits
  • Dropout — randomly disable neurons during training (for neural networks)
  • Early stopping — stop training when validation error starts rising
  • Simpler model — reduce the number of features or parameters
Opposite problem — underfitting
Underfitting is when the model is too simple to capture the pattern — poor on both training AND test data. The goal is the sweet spot between the two, called the bias-variance tradeoff.
Mid Explain the bias-variance tradeoff. ▶
Answer
  • Bias — error from wrong assumptions. High bias = model too simple = underfitting.
  • Variance — error from sensitivity to small fluctuations in training data. High variance = model too complex = overfitting.

The tradeoff: as you make a model more complex, bias decreases but variance increases. The goal is to find the balance that minimises total error on unseen data.

Simple analogy for the interview
High bias is like a student who memorised one formula and applies it to everything. High variance is like a student who memorised every past exam answer word-for-word but can't handle a slightly different question. You want the student who understood the concepts.
Model evaluation — critical for your thesis topic
Mid Explain precision, recall, and F1 score. When does each matter? ▶
Answer
  • Precision — of all the things the model flagged as positive, how many were actually positive? = TP / (TP + FP)
  • Recall — of all the actual positives, how many did the model catch? = TP / (TP + FN)
  • F1 score — the harmonic mean of precision and recall. Use when you need a single balanced metric.
When each matters
  • Recall matters most when missing a positive is costly — e.g. intrusion detection (missing an attack is dangerous) or cancer screening (missing a tumour is fatal).
  • Precision matters most when false alarms are costly — e.g. spam filters (blocking a real email is annoying).
Your thesis is the perfect example
"In my intrusion detection thesis, recall was critical — a missed attack (false negative) could mean a real security breach. But precision mattered too, because too many false alarms would overwhelm security analysts. I used F1 score to balance both, and reported the confusion matrix to show the tradeoff clearly."
Mid Why is accuracy a bad metric for imbalanced datasets? ▶
Answer

If 99% of network traffic is normal and 1% is attacks, a model that predicts "normal" for everything gets 99% accuracy — while catching zero attacks. Accuracy looks great but the model is useless.

For imbalanced data, use precision, recall, F1, and the confusion matrix instead. These reveal how the model performs on the minority class that actually matters.

Directly relevant to your work
"Intrusion detection datasets are heavily imbalanced — attacks are rare compared to normal traffic. That's exactly why I focused on recall and F1 rather than raw accuracy, and used techniques like resampling to handle the class imbalance."
Mid How do you handle imbalanced datasets? ▶
Answer
  • Oversampling the minority class — e.g. SMOTE, which generates synthetic examples of the rare class
  • Undersampling the majority class — remove some majority examples
  • Class weights — penalise mistakes on the minority class more heavily during training
  • Use the right metrics — precision, recall, F1 instead of accuracy
  • Anomaly detection approach — treat the rare class as anomalies rather than a classification problem
02 Deep learning & neural networks
Neural network fundamentals
Mid What is a neural network and how does it learn? ▶
Answer

A neural network is layers of interconnected nodes (neurons). Each connection has a weight. Data flows through: input layer → hidden layers → output layer.

How it learns:

  • Forward pass — data flows through, producing a prediction
  • Loss calculation — compare prediction to actual answer
  • Backpropagation — calculate how much each weight contributed to the error
  • Gradient descent — adjust weights to reduce the error
  • Repeat thousands of times until the error is minimised
Mid What is an activation function and why do we need it? ▶
Answer

An activation function decides whether a neuron should "fire" and introduces non-linearity into the network. Without it, no matter how many layers you stack, the network could only learn linear relationships.

Common ones:

  • ReLU — most common. Outputs the input if positive, else zero. Fast and avoids vanishing gradients.
  • Sigmoid — squashes output to 0-1. Used for binary classification output.
  • Softmax — converts outputs to probabilities that sum to 1. Used for multi-class classification.
  • Tanh — squashes to -1 to 1.
Senior What is the difference between CNN, RNN, and Transformer architectures? ▶
Answer
  • CNN (Convolutional Neural Network) — designed for spatial data like images. Uses filters to detect features (edges, shapes). Your OpenCV work relates to this domain.
  • RNN (Recurrent Neural Network) — designed for sequential data like time series or text. Has memory of previous inputs. LSTM and GRU are improved variants that handle long sequences.
  • Transformer — the modern architecture behind ChatGPT and most current AI. Uses "attention" to weigh the importance of different parts of the input. Replaced RNNs for most language tasks because it processes sequences in parallel.
If asked about LLMs
Large Language Models like GPT are built on the Transformer architecture. The key innovation is the "self-attention" mechanism, which lets the model focus on relevant words regardless of their position in the sentence. You don't need deep expertise — just understand the high-level concept.

Answer — the standard end-to-end flow

  • 10. Deployment & monitoring — ship it and watch for drift

Mid How do you handle missing data? ▶
Answer
  • Remove — drop rows/columns if missing data is small and random
  • Mean/median imputation — fill numeric gaps with the average (median is more robust to outliers)
  • Mode imputation — fill categorical gaps with the most common value
  • Forward/backward fill — for time series data
  • Predictive imputation — train a model to predict the missing values
  • Flag it — sometimes "missing" is itself meaningful, so add an indicator column
Key point interviewers want
Always understand WHY data is missing before deciding how to handle it. Missing completely at random is different from missing for a systematic reason. Blindly filling with the mean can introduce bias.
Mid What is feature engineering and why is it important? ▶
Answer

Feature engineering is creating, transforming, or selecting the input variables that a model learns from. It's often the difference between a mediocre model and a great one — "garbage in, garbage out."

Common techniques:

  • Scaling/normalisation — bring features to the same range so no single feature dominates
  • Encoding — convert categories to numbers (one-hot encoding, label encoding)
  • Creating new features — e.g. deriving "packets per second" from raw packet counts and time
  • Feature selection — remove irrelevant or redundant features to reduce noise
Connect to your work
"In my thesis, feature selection was important — the network traffic dataset had many features, and selecting the most discriminative ones improved both accuracy and training speed."
Senior What is the difference between L1 and L2 regularisation? ▶
Answer
  • L1 (Lasso) — adds the absolute value of weights as a penalty. Can shrink some weights to exactly zero, effectively performing feature selection. Use when you suspect many features are irrelevant.
  • L2 (Ridge) — adds the squared value of weights as a penalty. Shrinks weights toward zero but rarely exactly zero. Use when all features are somewhat useful.

Both prevent overfitting by discouraging large weights. Elastic Net combines both.

Mid What is a confusion matrix? ▶
Answer

A confusion matrix is a table showing how a classification model's predictions compare to reality:

  • True Positive (TP) — predicted attack, was actually an attack ✓
  • True Negative (TN) — predicted normal, was actually normal ✓
  • False Positive (FP) — predicted attack, was actually normal (false alarm)
  • False Negative (FN) — predicted normal, was actually an attack (missed threat)

All the key metrics — precision, recall, accuracy, F1 — are calculated from these four numbers.

You used this in your thesis
"I used confusion matrices to evaluate my intrusion detection models — they made the false negative rate visible, which was the most important number since a false negative means a missed attack."
Senior How would you deploy and monitor a machine learning model in production? ▶
Answer — this connects your ML and software engineering skills
  • Wrap in an API — expose the model behind a REST endpoint (you can do this with .NET or Python/Flask)
  • Containerise — package with Docker for consistent deployment
  • Version everything — model version, data version, code version
  • Monitor for data drift — the real-world data distribution changes over time, degrading model performance
  • Monitor performance metrics — track precision/recall on live data, not just at training time
  • Retraining pipeline — schedule periodic retraining on fresh data
  • A/B testing — compare new model versions against the current one before full rollout
