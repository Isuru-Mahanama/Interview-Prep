01Cloud & DevOps
Containers & orchestration
CoreWhat is Docker and why is it useful?▶

Docker packages an application and all its dependencies into a container — a lightweight, portable unit that runs identically everywhere. "It works on my machine" becomes "it works everywhere".

Why it matters: consistent environments across dev/test/production, faster deployment, isolation between apps, and efficient resource use compared to full virtual machines.

Simple explanation
A container is like a shipping container — it doesn't matter what's inside or what ship carries it, the container has a standard interface. Your app runs the same on your laptop, a colleague's machine, and the production server.
MidWhat is Kubernetes and when do you need it?▶

Kubernetes (K8s) is a container orchestration platform. When you have many containers across many servers, it handles: deployment, scaling up/down based on load, self-healing (restarting failed containers), load balancing, and rolling updates with zero downtime.

When you need it: when running microservices at scale. For a small app, it's overkill — Docker alone or a simple cloud service is enough.

CI/CD & automation
CoreWhat is CI/CD?▶
  • CI (Continuous Integration) — developers merge code frequently, and each merge automatically triggers a build and runs tests. Catches bugs early.
  • CD (Continuous Delivery/Deployment) — code that passes tests is automatically prepared for release (delivery) or actually deployed to production (deployment).

Tools: GitHub Actions, Azure DevOps Pipelines, Jenkins, GitLab CI.

Your experience
"At Mainframe we used Azure DevOps for our CI/CD pipelines — every commit triggered automated builds and tests before deployment, which caught issues before they reached production."
MidWhat is Infrastructure as Code (IaC)?▶

IaC means managing your infrastructure (servers, networks, databases) through code and configuration files rather than manual clicking in a console. Tools: Terraform, Azure Bicep, AWS CloudFormation.

Benefits: reproducible environments, version-controlled infrastructure, easy to spin up identical staging/production, and no "it worked when I set it up manually" problems.

Cloud platforms
CoreWhat are the main cloud service models (IaaS, PaaS, SaaS)?▶
  • IaaS (Infrastructure as a Service) — you rent raw servers/storage. You manage the OS and everything above. e.g. Azure VMs, AWS EC2.
  • PaaS (Platform as a Service) — you deploy your code, the platform manages servers, scaling, OS. e.g. Azure App Service, Heroku.
  • SaaS (Software as a Service) — fully managed software you just use. e.g. Gmail, Office 365.
MidWhat is serverless computing?▶

Serverless means you write functions that run on-demand without managing any servers. The cloud provider handles all infrastructure and scaling, and you only pay when your code runs. e.g. Azure Functions, AWS Lambda.

Good for: event-driven tasks, APIs with variable traffic, background jobs. Not good for: long-running processes or apps needing consistent low latency (cold starts).

02Modern development practices
APIs & architecture
MidREST vs GraphQL — what's the difference?▶
  • REST — multiple endpoints, each returns a fixed data structure. Simple, cacheable, widely used. Can over-fetch (get more data than needed) or under-fetch (need multiple calls).
  • GraphQL — single endpoint, the client specifies exactly what data it wants. Solves over/under-fetching. More complex to set up and cache.

Use REST for most APIs; GraphQL when clients need flexible queries or you have many different frontend needs (mobile vs web).

MidWhat are WebSockets and when do you use them?▶

WebSockets provide a persistent, two-way connection between client and server. Unlike HTTP (request-response), the server can push data to the client anytime.

Use for: real-time features — live chat, notifications, live sports scores, collaborative editing, and live odds updates in a betting platform.

Connect to your domain
"For real-time odds in a betting platform, WebSockets are ideal — instead of the client polling for updates every second, the server pushes new odds the moment they change."
Testing & quality
CoreExplain the testing pyramid▶

The testing pyramid describes the ideal balance of test types:

  • Unit tests (base, most numerous) — test individual functions in isolation. Fast, cheap.
  • Integration tests (middle) — test how components work together, e.g. API + database.
  • End-to-end tests (top, fewest) — test the whole flow like a user would. Slow, expensive, but valuable.

The principle: many fast unit tests, fewer slow E2E tests. An inverted pyramid (mostly E2E) is a common anti-pattern — slow and brittle.

Version control
MidExplain Git branching strategies▶
  • Feature branching — each feature developed in its own branch, merged via pull request after review
  • Git Flow — structured branches: main, develop, feature, release, hotfix. Good for scheduled releases
  • Trunk-based — everyone commits to main frequently behind feature flags. Favoured by big tech for continuous deployment

Common commands to know: merge vs rebase, cherry-pick, resolving conflicts, reverting.

03AI in modern development

Increasingly asked in 2025+ interviews. Companies want engineers who use AI tools effectively and understand their limits.

TrendingHow do you use AI coding assistants responsibly?▶

A strong answer shows you use tools like GitHub Copilot or Claude to boost productivity but with judgment:

  • Use them to accelerate boilerplate, tests, and exploring unfamiliar APIs
  • Always review and understand generated code — never ship what you don't understand
  • Be aware of security and licensing concerns with generated code
  • They augment your skills, they don't replace understanding fundamentals
The balanced answer
"I use AI assistants to move faster on routine tasks, but I treat their output like a junior developer's PR — I review everything before it goes in. The fundamentals still matter; the tools just remove friction."
TrendingWhat is RAG (Retrieval-Augmented Generation)?▶

RAG is a technique to make LLMs more accurate by giving them relevant external information at query time. Instead of relying only on training data, the system retrieves relevant documents (from a database or knowledge base) and includes them in the prompt.

Why it matters: reduces hallucinations, lets the model answer about private/current data it wasn't trained on. It's how most enterprise AI chatbots work today.

TrendingWhat is an API rate limit and why do LLM APIs have them?▶

Rate limits cap how many requests you can make in a time window. LLM APIs use them to manage compute costs and ensure fair usage. As a developer integrating these, you handle them with retry logic, exponential backoff, and caching responses where possible.

04Security fundamentals
MidWhat are common web security vulnerabilities?▶
  • SQL Injection — malicious SQL in inputs. Prevent with parameterised queries (which EF Core does by default)
  • XSS (Cross-Site Scripting) — injecting malicious scripts. Prevent by escaping output and using Content Security Policy
  • CSRF (Cross-Site Request Forgery) — tricking a user into unwanted actions. Prevent with anti-forgery tokens
  • Broken authentication — weak session/password handling. Use proven auth like JWT with proper validation

These are the OWASP Top 10 basics — worth knowing given your security background.

Your edge
"With my intrusion detection thesis background, I'm particularly conscious of security — I understand both the attack side and how to defend against it in application code."
